Question 1️⃣: What are Functions and Different Ways to Create Them?

💬 Interviewer: "Explain JavaScript functions and show me 3 different ways to create them."
Answer: A function is a reusable block of code that performs a specific task. Think of it like a recipe - you write it once, then use it whenever you need it!
🏠 Real-world analogy: A function is like a coffee machine. You put in ingredients (parameters), press a button (call the function), and get coffee (return value). You don't need to know HOW it works inside, just how to use it!
/* ============================================ METHOD 1: FUNCTION DECLARATION - Most common way - Can be called before it's defined (hoisting) ============================================ */ function greet(name) { // Parameters: name (input) return "Hello, " + name + "!"; // Return value (output) } // Calling the function console.log(greet("Sarah")); // Output: "Hello, Sarah!" /* ============================================ METHOD 2: FUNCTION EXPRESSION - Function assigned to a variable - CANNOT be called before definition ============================================ */ let sayGoodbye = function(name) { return "Goodbye, " + name + "!"; }; console.log(sayGoodbye("John")); // Output: "Goodbye, John!" /* ============================================ METHOD 3: ARROW FUNCTION (ES6+) - Shorter syntax - Modern JavaScript way ============================================ */ let multiply = (a, b) => { return a * b; }; // Even shorter (one-liner, implicit return) let add = (a, b) => a + b; console.log(multiply(5, 3)); // Output: 15 console.log(add(10, 20)); // Output: 30

Question 2️⃣: What is Function Scope?

💬 Interviewer: "Explain function scope with an example. What's the difference between local and global variables?"
Answer: Scope determines where variables can be accessed. Variables defined inside a function are LOCAL (only accessible inside that function). Variables defined outside are GLOBAL (accessible everywhere).
🏠 Real-world analogy: Think of scope like rooms in a house.
  • Global scope: The living room - everyone can access it
  • Local scope: Your bedroom - only you can access your private stuff
  • You can see out from your bedroom to the living room, but people in the living room can't see into your bedroom!
/* ============================================ SCOPE EXAMPLE ============================================ */ // GLOBAL VARIABLE - Accessible everywhere let globalVar = "I am global!"; function scopeDemo() { // LOCAL VARIABLE - Only accessible inside this function let localVar = "I am local!"; console.log(globalVar); // ✅ Works! Can access global from inside console.log(localVar); // ✅ Works! Inside the function } scopeDemo(); console.log(globalVar); // ✅ Works! Global is accessible everywhere console.log(localVar); // ❌ ERROR! localVar doesn't exist here /* ============================================ NESTED SCOPE (Functions inside functions) ============================================ */ function outer() { let outerVar = "I'm from outer"; function inner() { let innerVar = "I'm from inner"; // Inner can access outer's variables console.log(outerVar); // ✅ Works! console.log(innerVar); // ✅ Works! } inner(); console.log(outerVar); // ✅ Works! console.log(innerVar); // ❌ ERROR! Can't access inner from outer } outer();

🔑 Key Rule:

Inner functions can see outer variables, but outer functions CANNOT see inner variables.

It's like a one-way mirror - you can look out, but people can't look in!

Question 3️⃣: What are Closures? (Most Important!)

💬 Interviewer: "Explain closures and show me a practical example."
Answer: A closure is when an inner function "remembers" variables from its outer function, even AFTER the outer function has finished running. It's like giving the inner function a backpack with the outer variables inside!
🎒 Real-world analogy: Imagine you go to college (outer function). While there, you make friends and learn skills (variables). When you graduate and leave college (outer function ends), you still REMEMBER your friends and skills (closure). The memories come with you wherever you go!
/* ============================================ CLOSURE EXAMPLE 1: Simple Counter ============================================ */ function createCounter() { // This variable is "private" - only accessible through the returned function let count = 0; // Return an inner function return function() { count++; // This remembers 'count' even after createCounter finishes! return count; }; } // Create a counter let counter = createCounter(); // Each time we call it, it remembers the count! console.log(counter()); // Output: 1 console.log(counter()); // Output: 2 console.log(counter()); // Output: 3 // count variable is NOT accessible directly console.log(count); // ❌ ERROR! count is private /* ============================================ CLOSURE EXAMPLE 2: Personalized Greeting ============================================ */ function makeGreeting(greeting) { // 'greeting' is captured by the closure return function(name) { // This function remembers 'greeting' from outer function return greeting + ", " + name + "!"; }; } // Create different greeting functions let sayHello = makeGreeting("Hello"); let sayHi = makeGreeting("Hi"); console.log(sayHello("Alice")); // Output: "Hello, Alice!" console.log(sayHi("Bob")); // Output: "Hi, Bob!" // Each closure has its own 'greeting' value! /* ============================================ CLOSURE EXAMPLE 3: Private Variables ============================================ */ function createBankAccount(initialBalance) { // Private variable - can't be accessed directly let balance = initialBalance; // Return object with methods (closure!) return { deposit: function(amount) { balance += amount; return "New balance: $" + balance; }, withdraw: function(amount) { if (amount > balance) { return "Insufficient funds!"; } balance -= amount; return "New balance: $" + balance; }, getBalance: function() { return "Current balance: $" + balance; } }; } let myAccount = createBankAccount(100); console.log(myAccount.getBalance()); // "Current balance: $100" console.log(myAccount.deposit(50)); // "New balance: $150" console.log(myAccount.withdraw(30)); // "New balance: $120" // Can't access balance directly (it's private!) console.log(myAccount.balance); // undefined

🔑 Why Closures Matter:

  • Data Privacy: Keep variables private (encapsulation)
  • Memory: Functions remember their environment
  • Factory Functions: Create multiple instances with their own data

Question 4️⃣: What is an IIFE?

💬 Interviewer: "What is an IIFE and why would you use it?"
Answer: IIFE stands for "Immediately Invoked Function Expression". It's a function that runs as soon as it's defined. You create it and execute it in one step!
🚀 Real-world analogy: An IIFE is like a self-destructing message. You write it, it does its job immediately, and then it's gone - no trace left behind. Perfect for one-time tasks!
/* ============================================ IIFE SYNTAX ============================================ */ // Regular function (you call it manually) function regularFunction() { console.log("I need to be called!"); } regularFunction(); // You call it // IIFE (calls itself immediately) (function() { console.log("I run immediately!"); })(); // Notice the () at the end - it calls itself! /* ============================================ IIFE BREAKDOWN ============================================ */ ( // Step 1: Wrap in parentheses function() { // Step 2: Create function console.log("IIFE!"); } ) // Step 3: Close parentheses (); // Step 4: Call it immediately! /* ============================================ WHY USE IIFE? 1. Avoid polluting global scope 2. Create private variables 3. Run code once immediately ============================================ */ // Example 1: Private variables (function() { let privateVar = "I'm private!"; console.log(privateVar); // Works inside })(); console.log(privateVar); // ❌ ERROR! Not accessible outside // Example 2: With parameters (function(name) { console.log("Hello, " + name + "!"); })("World"); // Pass "World" as argument // Example 3: Arrow function IIFE (modern way) (() => { console.log("Arrow IIFE!"); })(); /* ============================================ PRACTICAL USE CASE: Initialize app ============================================ */ (function() { // This code runs once when page loads let appName = "My App"; let version = "1.0"; console.log(appName + " v" + version + " initialized!"); // Setup code here... // These variables are private and won't conflict with other code })(); /* ============================================ IIFE RETURNING VALUE ============================================ */ let result = (function() { let x = 10; let y = 20; return x + y; // Returns 30 })(); console.log(result); // Output: 30

🔑 When to Use IIFE:

  • One-time initialization: Run setup code once
  • Avoid global variables: Keep scope clean
  • Module pattern: Create private/public methods
  • Loops with closures: Capture loop variables correctly

Question 5️⃣: Classic Interview Problem - Loop with Closure

💬 Interviewer: "Why does this code print '3' three times instead of 0, 1, 2? How do you fix it?"
/* ============================================ THE PROBLEM (var has function scope!) ============================================ */ for (var i = 0; i < 3; i++) { setTimeout(function() { console.log(i); // What will this print? }, 1000); } // Expected: 0, 1, 2 // Actual: 3, 3, 3 ❌ Why?! // REASON: By the time setTimeout runs (after 1 second), // the loop has finished and i = 3 for all callbacks! /* ============================================ SOLUTION 1: Use 'let' (block scope) ============================================ */ for (let i = 0; i < 3; i++) { setTimeout(function() { console.log(i); // ✅ Prints: 0, 1, 2 }, 1000); } // 'let' creates a new 'i' for each iteration! /* ============================================ SOLUTION 2: Use IIFE to capture value ============================================ */ for (var i = 0; i < 3; i++) { (function(capturedI) { setTimeout(function() { console.log(capturedI); // ✅ Prints: 0, 1, 2 }, 1000); })(i); // Pass current i to IIFE } // IIFE creates a closure that captures the current value!

🔑 Key Takeaway:

var has function scope → shared across all iterations

let has block scope → new variable for each iteration

IIFE creates closure → captures value at that moment

📋 Quick Summary

Functions:

• Reusable blocks of code

• 3 ways: declaration, expression, arrow

Scope:

• Global: accessible everywhere

• Local: only inside function

• Inner can see outer, but not vice versa

Closures:

• Inner function remembers outer variables

• Even after outer function finishes

• Creates private variables

IIFE:

• Immediately Invoked Function Expression

• Runs once immediately

• Syntax: (function() { ... })();

• Keeps global scope clean
